# Week 8: 结课项目

恭喜你来到课程的最后一周！🎉

## 📋 项目概述

Week 8 是结课项目周，你将运用前 7 周学到的所有知识，独立完成一个完整的 Cardano Haskell 应用。

这不仅是一次技能检验，更是一次**创造的机会**——你将构建一个真实可用的工具！

## 🎯 学习目标

通过完成结课项目，你将：

1. **综合应用** Weeks 1-7 的所有知识
   - Haskell 基础语法和类型系统
   - Monad 和 IO 操作
   - 模块化项目组织
   - 错误处理和测试
   - JSON 解析和 API 调用
   - Cardano 数据结构处理

2. **独立解决问题**
   - 理解需求并分解任务
   - 设计模块结构
   - 调试和测试代码
   - 优化用户体验

3. **完成真实项目**
   - 从零到完整应用
   - 专业的代码组织
   - 完整的文档
   - 可演示的成果

## ⏱️ 时间估计

- **总时间**：8-10 小时
  - 规划和设计：1 小时
  - 核心功能实现：4-5 小时
  - 测试和调试：1-2 小时
  - 优化和完善：1-2 小时
  - 文档和演示准备：1 小时

**建议分配**：
- 第 1-2 天：规划 + 基础结构 + 核心功能（50%）
- 第 3-4 天：完成功能 + 测试（80%）
- 第 5 天：优化 + 文档 + 演示准备（100%）

## 🎨 项目选择（二选一）

你需要从以下两个项目中**选择一个**完成：

### 项目 1: 命令行钱包工具 💼

**描述**：构建一个命令行 Cardano 钱包工具，支持地址管理、余额查询和交易构建。

**适合人群**：
- 喜欢工具开发
- 对交易感兴趣
- 希望深入理解 Cardano 交易结构

**核心功能**：
- ✅ 地址生成和管理
- ✅ 余额查询（Blockfrost API）
- ✅ UTxO 查看
- ✅ 简单支付交易构建（模拟）
- ✅ 钱包状态保存/加载
- ✅ 多命令 CLI 界面

**技术栈**：
- aeson (JSON 处理)
- req (API 调用)
- bytestring (数据处理)
- ExceptT (错误处理)
- Hspec (测试)

**难度**：⭐⭐⭐☆☆ (中等)

**详细规格** → [项目 1 规格书](project-1-wallet.md)

---

### 项目 2: Cardano 余额监控器 📊

**描述**：构建一个自动化工具，定期检查 Cardano 地址余额变化并发送通知。

**适合人群**：
- 喜欢自动化工具
- 对数据追踪感兴趣
- 希望构建实用的监控系统

**核心功能**：
- ✅ 添加/删除监控地址
- ✅ 定期余额检查
- ✅ 余额变化检测和记录
- ✅ 控制台通知
- ✅ 历史数据持久化
- ✅ 配置文件管理

**技术栈**：
- aeson (JSON 和配置)
- req (API 调用)
- time (定时任务)
- ExceptT (错误处理)
- file I/O (数据存储)

**难度**：⭐⭐⭐☆☆ (中等)

**详细规格** → [项目 2 规格书](project-2-monitor.md)

## 📊 评估标准

你的项目将从以下方面评估：

| 维度 | 权重 | 说明 |
|------|------|------|
| **功能完整性** | 60% | 所有必需功能正常工作 |
| **代码质量** | 20% | 组织清晰、命名合理、有类型签名 |
| **测试** | 10% | 有基础测试或详细的测试文档 |
| **用户体验** | 10% | 清晰的帮助信息、友好的错误提示 |

**及格线**：70 分（实现所有必需功能 + 基本代码质量）

**优秀线**：85 分（功能完善 + 代码优秀 + 有测试 + UX 好）

详细评分标准见 [evaluation.md](evaluation.md)

## 🚀 开始步骤

### 第 1 步：选择项目
花 15 分钟阅读两个项目规格书，选择更感兴趣的一个。

### 第 2 步：阅读指南
阅读 [implementation-guide.md](guide.md)，了解实施技巧。

### 第 3 步：设置环境
```bash
cd exercises/week-08/projects/wallet-tool  # 或 balance-monitor
cabal build
```

### 第 4 步：制定计划
打开 `TASKS.md`，规划你的实现顺序。

### 第 5 步：开始编码
从最核心的功能开始，逐步完善。

### 第 6 步：测试和调试
边写边测，确保每个功能都正常工作。

### 第 7 步：文档和演示
完善 README，准备演示内容。

## 💡 实施建议

### 时间管理
- **不要追求完美**：先完成必需功能，再考虑优化
- **增量开发**：每次实现一个小功能，立即测试
- **留出缓冲**：最后一天用于处理意外问题

### 技术建议
- **复用 Week 7 代码**：许多模式可以直接应用
- **使用提供的工具**：`common/` 目录有现成的辅助函数
- **参考示例**：Week 5-7 的练习是很好的参考

### 求助指南
- **查阅讲义**：Weeks 1-7 的讲义是最好的参考
- **运行示例**：Week 5-7 的 examples 目录有可运行的代码
- **调试技巧**：使用 GHCi、print 调试、类型驱动开发
